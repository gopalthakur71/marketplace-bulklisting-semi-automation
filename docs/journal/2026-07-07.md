## The Fix-Error flow (what it does)

Reads **three input formats** — per-SKU rejection xlsx, file-level csv, and the MDirect Listings Report
— via a fingerprint detector. Every error is explained in plain English through a strict lookup order:
**curated YAML rules → learned store → Gemini (explain-only) → raw fallback**. Then it splits issues
into two groups and gates on a human choice:

- **Surface A (correct-in-place):** deterministic auto-fixes (brand, address/Manufacturer+Packer, etc.)
  applied to the rejected rows, behind a **Proceed / Do-not-change** gate.
- **Surface B (rebuild):** rejected SKUs regenerated through the pipeline, re-pinning each SKU's stored
  styleGroupId + HSN from the registry.

A drop-SKU checkbox is authoritative on both surfaces; a Phase-D append-only **correction log** leaves a
breadcrumb of every change.

## Real bugs the reviews caught (each would have shipped)

1. **`read_only=True` misreads dimensions on real Myntra xlsx.** openpyxl trusts a stale `<dimension>`
   tag in the file, so streaming reads dropped rows/columns. Fix: dropped `read_only` + added a
   **zip-patch regression test** that corrupts the dimension tag and asserts we still read every cell.
   (`629cd32`) — the single most dangerous one; it silently loses data.
2. **Whole-sheet rebuild lost its registry pins.** `regenerate_surface_b` with `skus=None` resolved
   **no** registry pins → brand-new styleGroupIds + lost HSN on a full rebuild. Fix: iterate
   `reg.keys()` when `skus is None` + regression test. (`a0c02e5`)
3. **Empty rebuild set triggered a whole-catalog rebuild.** An empty Surface-B set was falsy and fell
   through to "rebuild everything". Fix: distinguish empty-set from None. (`5a3aff9`)
4. **Explain-only Surface-B issues silently lost.** A `setdefault`/`manual_needed` trap in `fix.py`
   dropped them; changed to `summary.get("manual_needed") or [...]`.

## The deploy — full backlog to prod

Everything since 2026-07-02 sat unmerged on feature branches. Decision (Gopal, option A): **full prod
setup, then deploy** the whole stack at once. Executed in order:

1. **Local-dev polish, committed.** `load_settings()` now auto-loads a repo-root `.env` via
   `load_dotenv(override=False)` when no explicit env dict is passed — real env/SSM always win, no-op in
   prod (no `.env` shipped) and in tests. `.dockerignore` blocks `.env`/`.env.*` from the image. (`d76b3bf`)
2. **Gemini key → SSM.** `/marketplace-listing/gemini_api_key` as a **SecureString** (ap-south-1). It's
   in settings' `_FIELDS`, so prod reads it from SSM with `WithDecryption=True` — same pattern as the
   Cognito client secret. Gopal added it via console. Runbook: `docs/runbooks/add-gemini-api-key-ssm.md`.
3. **`EXPLAIN_WITH_GEMINI=1` on the box.** This switch + `GEMINI_MODEL` are **env-only** (not in SSM),
   living in the `docker run -e` lines of the systemd unit. Rewrote the **live**
   `/etc/systemd/system/listing-app.service` via Session Manager (a plain `systemctl restart` — what CI
   runs — reuses the old unit, so it had to be rewritten) → verified `EXPLAIN_WITH_GEMINI=1` + `active`.
   Also baked into repo `aws/ec2/userdata.sh` (source-of-truth). Runbook:
   `docs/runbooks/enable-gemini-ec2-systemd.md`. (`11b63f8`)
4. **EC2 box powered on.** The deploy job filters `instance-state-name=running`, so the box must be up
   or the deploy job goes red (image still lands in ECR; self-heals next boot).
5. **Merge + push.** Full suite **160 passing** locally → clean fast-forward `09064ec..ce75146` →
   `git push origin main` → CI/CD (test → build-and-push `:latest` → deploy via SSM restart).

### Config-drift note (this deploy's known gap)
Three layers, don't conflate: the **repo** `userdata.sh` (updated, source-of-truth), the instance's
**stored user-data** attribute (NOT updated — stale but harmless; it only runs on a *first* boot, not a
restart), and the **live** unit file (rewritten). If this box is ever rebuilt, launch it from the
updated `userdata.sh`. This is a live patch, and the drift is tracked deliberately.

### Gotcha worth remembering
The IAM user present in the local Git-Bash shell (`Meta-ad-Banner`) **lacks EC2/SSM privileges**, so
all AWS writes were done by Gopal via the console — not from the CLI here. Also `gh` is not installed
locally, so the CI run is watched on the GitHub Actions web page, not streamed.
