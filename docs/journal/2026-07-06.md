## Key model decisions (Gopal's calls)

1. **HSN is authoritative per-SKU.** The registry pins the exact HSN used for each SKU. The
   `category|fabric` signature KB is **suggestion-only** — it proposes on the review screen but is
   never deterministic for HSN. This avoids a wrong learned code silently propagating.
2. **App-owned registry, not the Myntra report.** Change detection = hash of the final Myntra output
   row (excluding `styleGroupId` + HSN). Grounding fact that shaped this: **Myntra keys a listing on
   the seller SKU code** (= our Shopify SKU), and assigns its own `style id`/`sku id`. Our
   `styleGroupId` is just a per-sheet grouping requirement, **not** the listing key — learned from a
   full live-catalog `MDirect_Listings_Report` export.
3. **The guard triggers on the 2nd upload, by design.** The 1st generate is what *records* the SKU
   (no warning, correct); re-uploading the same file then warns. Gopal hit this and it's intended.

## The three UI gotchas (found by running the real app)

1. **A `required` input inside a `display:none` div silently blocks the whole form.** The B.4
   styleGroupId "Edit" partial nested `<input required>` hidden in the Generate form → clicking
   **Generate** ran HTML5 validation, couldn't focus the hidden required field, and **aborted the
   submit with no request sent** (console: "An invalid form control … is not focusable"). Symptom to
   recognise: a button "does nothing" and there's **no POST in the uvicorn log**. Fix: never put
   `required` on a hidden field; give in-form htmx buttons explicit `type="button"`. (`4063029`)
2. **Static assets must be cache-busted or edits never show.** After a CSS edit the page kept
   rendering old styles (no `GET /static/app.css` in the log = served from cache). Fix: `main.py`
   registers a Jinja global `asset_v(filename)` = the file's mtime; `base.html` links
   `app.css?v={{ asset_v('app.css') }}`. Every change now busts cache automatically. (`4511272`)
3. **`<pre>` overflows its panel.** The Phase-1 fill report's long `[BLANK] …` lines spilled past the
   box under the browser default `white-space:pre`. Fix: `pre{white-space:pre-wrap;
   overflow-wrap:anywhere;max-width:100%}`. (`3006178`)
