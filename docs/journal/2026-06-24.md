## Errors encountered & resolutions (chronological)

1. **"No rows filled / file shows old data."**
   Cause: `output/myntra_filled.xlsx` was open in Excel, so `run.py` couldn't
   overwrite it (PermissionError). Resolution: close the file before each run.
   (Recurred several times — same fix.)

2. **Images not uploading (local filenames in image columns).**
   Cause: image columns held local names like `SKU_1.jpg`; Myntra ingests images
   by URL. Resolution: write the Shopify **CDN URLs** into the image columns.
   Note: Shopify CDN serves JPEG to standard clients even though the URL ends in
   `.webp` (content negotiation). Per-SKU folder fallback (cells = bare filename)
   kept in reserve.

3. **`SHEET_VALIDATION_FAILED` — null brand (getBrandCodeFromBrandName).**
   This single symptom had **three** distinct underlying causes, peeled back one
   at a time. The message ("null brand") was misleading — it actually meant
   Myntra could not read the sheet at all, so every field came back null.

   a. **Re-injected dropdown validations.** We hand-edited the xlsx zip to put
      the 37 x14 dropdowns back; Myntra's Apache POI parser rejected the injected
      extension XML. Evidence: the known-good file has **no dropdowns**.
      Resolution: stop injecting dropdowns into the upload file
      (`preserve_dropdowns=False`; opt-in only for a manual-edit copy).

   b. **Shared strings.** openpyxl stores text as shared strings (cells hold an
      index into a table); Myntra's parser doesn't resolve them, so even the
      column headers read null → it couldn't locate the brand column. The good
      file uses **inline strings**. Resolution: post-save, convert the Sarees
      sheet's `t="s"` cells to `t="inlineStr"` with embedded text.

   c. **Leftover template example rows (the real root cause).** The blank Myntra
      template ships with stray example image URLs in **row 11** (hyperlinked
      cells, no brand). We only wrote products to rows 4–10 and never cleared the
      rest, so Myntra read row 11 as an 8th product with a null brand.
      Two openpyxl gotchas hid this:
      - `ws.cell(row, col, value=None)` is a **no-op** (None = "don't set") — the
        first clear attempt did nothing; must assign `cell.value = None`.
      - The stray cells were **hyperlinks**; openpyxl re-materialises the URL on
        save unless you also clear `cell.hyperlink`.
      Resolution: before writing, clear the whole data region
      (`first_data_row..max_row`) — both `.value` and `.hyperlink`. Verified:
      output now contains only rows 4–10, zero stray rows/hyperlinks.

4. **`SHEET_VALIDATION_FAILED` — Style SKU Count (CURRENT, open).**
   Message: *"Minimum unique StyleGroupIds required to process the sheet is 1.
   Given sheet has only 7."* This is **progress** — the file is now read and
   parsed; we've reached field-level validation. We assign sequential
   styleGroupIds 1–7 (one group per product). The message is oddly worded;
   the exact requirement is unclear (does Myntra want one shared group? a
   different id scheme? styleGroupId left blank?). **Unresolved — to investigate.**
