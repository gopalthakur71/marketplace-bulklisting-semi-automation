## Arc 2 — Error-corrector finalisation (2026-06-28)

`ce09257` — "honest auto-fixes + vocab-validated answers." Two principles cemented in the
corrector that later shaped the web Fix-flow UX:

1. **Honest auto-fixes** — the tool only claims to auto-fix what it can deterministically fix;
   everything else is surfaced for a human, not silently guessed.
2. **Vocab-validated answers** — any human-supplied corrected value is validated against
   Myntra's allowed dropdown vocabulary (`mapper.validate_value`) **before** it is written;
   invalid values are reported as *rejected*, never written. (See the standing rule
   "dropdown values must use the template's exact vocabulary spelling.")

This is the backend that Phase-2's Flow B drives.

The locked system:

  (Ingest → Map → Images→S3 → Fill&validate → Ready).

**Two flows confirmed:** (A) **Generate** a Myntra sheet from a Shopify `products_export.csv`;
(B) **Fix** rows Myntra rejected.

**Key UX decision (Gopal):** the Fix-errors flow takes a **free-text input where the user TYPES
the corrected value — NOT a multiple-choice option picker.** The typed value is still validated
against Myntra vocab before writing; a non-clickable "closest Myntra names" hint is OK, but no
option buttons.

**Other locked decisions (spec `a0d6bdc`):**
1. **Full-scope build** = FastAPI app + Cognito auth + SSM/Secrets config + Dockerfile; EC2
   deploy deferred to Phase 4.
2. **Layered config + `AUTH_DISABLED=1` toggle** — one settings loader reads env first, falls
   back to SSM/Secrets; build & test locally with NO AWS, then flip on Cognito, then
   SSM/Secrets, one at a time. (No EC2 needed to develop — the instance role is deploy-only.)
3. **Styling = reuse the hand-written mockup CSS (plain CSS), NOT Tailwind** — no Node, no
   build step, no runtime CDN. (This overrides earlier "Tailwind" wording.)
4. **Generate runs as a background job + htmx polling** stepper.
5. **Fix-errors corrections are typed free-text, validated against vocab** (see above).

Gopal's steer throughout: **keep it simple for now, no fancy design**; additions later.

**Plan (`b714d6b`, pre-flight fixes `ed73013`):** 9 TDD tasks + a final whole-branch review.

import-time AWS calls (made the AWS getters lazy + fail-soft so import never crashes offline).

### 3b. Build (2026-06-29 → 2026-06-30) via subagent-driven development

**Method:** for each task a *fresh implementer subagent* wrote code+tests (TDD), then an
*independent reviewer subagent* checked spec-compliance + code quality; fixers patched
Critical/Important findings; minor findings rolled up for the final review. Progress tracked in
`.superpowers/sdd/progress.md`. Implementers/reviewers ran on Sonnet (Haiku for the purely
mechanical ones); the final whole-branch review on Opus. Merge base = `c03a6e5`.

**The 9 tasks (all reviewed clean):**

1. **Settings loader + ledger store** (`6635d83`). `Settings` dataclass + `load_settings(env,
   ssm, secrets)` resolving each field env-first with **per-field** fallback to SSM (non-secret)
   / Secrets Manager (the Cognito client secret). `ledger_store()` returns a `LocalJsonStore`
   (dev) or the existing `S3JsonStore` (cloud). **Notable fix:** the first implementation gated
   AWS fallback all-or-nothing — if *any* non-secret env var was set, the secret silently never
   resolved. Reworked to true per-field fallback (lazy fail-soft getters, no gate). See Decisions.
2. **Auth** (`e48b942`). `current_user(settings, token)` returns a fixed dev user `dev@local`
   when `AUTH_DISABLED`, else validates a Cognito JWT (`verify_jwt`: RS256 pinned, audience +
   issuer checked, JWKS by `kid`, jose errors → `AuthError`). JWKS URL/issuer derive from the
   pool id + region.
3. **In-memory job store** (`f3033a2`). Thread-safe `JobStore` (create/get/set_step/finish/fail)
   + `Job` dataclass + the `STEPS` list, for the Generate background job + htmx polling.
4. **App factory + home page + static** (`68154b5`). `create_app()` (settings on `app.state`
   before routers; lazy router import; `AuthError` handler), `base.html` + `home.html`, the
   Marigold Ops `app.css`, vendored `htmx.min.js` (48 KB, real build).
5. **Flow A — Generate** (`c0bc7aa`). Upload CSV → **reserve** a styleGroupId batch → run the
   pipeline on a daemon thread → htmx-poll `/jobs/{id}` → download xlsx → **confirm** (advances
   the ledger). Invariant honoured: **`reserve()` does NOT advance the counter; only `confirm()`
   does.**
6. **Flow B — Fix** (`12a66b2`, hardened `572b3ee`). Upload the rejection .xlsx → classify into
   buckets (auto-fixed / needs-you / explain-only) → user **types** corrected values + ticks
   drop checkboxes → run `correct()` → download. Typed free-text, vocab-validated, no option
   buttons (per the locked UX decision).
7. **Dockerfile** (`8444761`). Swapped the `CMD` to `uvicorn src.web.main:app --host 0.0.0.0
   --port 8080` + `EXPOSE 8080`. (Docker daemon wasn't running on the Windows host, so the build
   smoke-test was skipped; the edit was verified directly.)
8. **Cognito console runbook** (`1d55cec`, corrected `36184ff`) —
   `docs/runbooks/web-cognito-setup-console.md`.
9. **SSM + Secrets console runbook** (`ec08a77`) —
   `docs/runbooks/web-ssm-secrets-setup-console.md`. SSM prefix `/marketplace-listing/`,

### 3c. Security findings in the Fix flow (caught & fixed — `572b3ee`)

An automated security review flagged the Task-6 commit. Two **HIGH** issues, both fixed:
- **Path traversal** — `fix_id` (a URL path param) was concatenated into a filesystem path with
  no validation. Fixed: validate `^[0-9a-f]{32}$` (it's always a `uuid4().hex`) → 404 otherwise,
  applied in *both* `fix_apply` and `fix_download`, plus a `realpath` containment check inside
  `RUNTIME` as defence-in-depth.
- **Unsafe deserialization** — session rows were `pickle`-dumped then `pickle.load`-ed (RCE
  vector when combined with the traversal). Fixed: dropped pickle entirely; serialize to JSON
  (`dataclasses.asdict`) and reconstruct `RowError(**d)`.

### 3d. Final whole-branch review (Opus) + closeout (2026-06-30)

## Deferred decisions / accepted risks

- **EC2 deploy (Phase 4).** The image is in ECR; actually running it on a server (instance role
  with `ssm:GetParameter*` / `secretsmanager:GetSecretValue`, the prod callback URL, fronting/
  TLS) is not done.
- **OAuth `/auth/callback` login flow — NOT built.** The app validates a JWT only if one is
  already present (cookie/header); there's no route that performs the Cognito hosted-UI redirect
  and sets the `id_token` cookie. Until then, run locally with `AUTH_DISABLED=1`. This is the
  first item of the deploy phase. (The Cognito runbook now says this plainly.)
- **IDOR on Fix sessions — accepted risk, deferred.** Fix sessions aren't scoped to the owner.
  Mitigated by a 128-bit unguessable random `fix_id` + a trusted internal team behind Cognito;
  consistent with "keep it simple." Revisit if the app ever opens to less-trusted users.
- **Minor code-quality follow-ups** (logged in `.superpowers/sdd/progress.md`): deprecated
  `TemplateResponse(name, ctx)` positional form used ~8× (migrate to `(request, name)`); a couple
  of double `get_user()` calls; the duplicated lazy `_templates()` import (could move to
  `app.state`); add a direct `verify_jwt` happy-path / unknown-kid test and a non-hex `fix_id`
  test; `current_user` falling back to `email="unknown"` could raise instead.
- **Full end-to-end Generate locally** wasn't run through the browser — the real pipeline would
  hit the "Images → S3" step, which needs real S3/image config. Offered to wire `upload=False`
  to demo a complete run without AWS; not done yet.
